# Git 커밋 컨벤션

현재 staged 변경사항을 분석하고 아래 컨벤션에 맞는 커밋 메시지를 작성한 뒤 커밋하세요.

## 커밋 타입

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 변경 |
| `style` | 코드 포맷팅 (기능 변경 없음) |
| `refactor` | 리팩토링 (기능·버그 변경 없음) |
| `test` | 테스트 추가·수정 |
| `chore` | 빌드·설정·패키지 등 기타 변경 |

## 형식

```
<type>: <subject>

[body]
```

- **subject**: 50자 이내, 한국어, 명령형, 마침표 없음
- **body**: 선택사항, 변경 이유나 상세 설명

## 절차

1. `git diff --staged` 로 변경사항 확인
2. 적절한 타입 선택
3. 커밋 메시지 작성 및 커밋 실행
