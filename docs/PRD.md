# 스터디 일지 공유 서비스 MVP PRD

## 핵심 정보

**목적**: Notion에 작성된 스터디 일지를 외부 클라이언트가 로그인 없이 토큰 URL로 열람하고 PDF로 저장할 수 있게 한다
**사용자**: 스터디 일지를 배포하는 관리자(어드민) 및 공유 링크로 일지를 열람하는 스터디 멤버/외부 열람자

---

## 사용자 여정

```
1. 어드민 - 랜딩 페이지
   ↓ "어드민 로그인" 버튼 클릭

2. 어드민 링크 생성 페이지 (패스코드 인증)
   ↓ Notion DB에서 일지 선택 + 만료일 설정 후 "링크 생성" 클릭

3. 토큰 URL 생성 완료 (/log/[token])
   ↓ 생성된 URL 복사 → 클라이언트에게 공유

─────────────────────────────────────

1. 클라이언트 - 공유 URL(/log/[token]) 접근
   ↓ 토큰 유효성 검사

   [토큰 유효 + 만료 전] → 스터디 일지 열람 페이지
   [토큰 무효 / 만료됨]  → 만료/오류 페이지

2. 스터디 일지 열람 페이지
   ↓ 일지 내용 확인

   [PDF 저장 원할 경우] → "PDF 다운로드" 버튼 클릭 → PDF 파일 저장
   [완료]               → 브라우저 종료
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|-------------|------------|
| **F001** | Notion 일지 조회 | Notion API로 스터디 일지 데이터를 읽어와 렌더링 | 서비스의 핵심 데이터 소스 | 스터디 일지 열람 페이지 |
| **F002** | 토큰 기반 공개 접근 | 고유 토큰(/log/[token])으로 로그인 없이 일지 열람 | 클라이언트 마찰 최소화 | 스터디 일지 열람 페이지 |
| **F003** | 만료일 검증 | 일지 접근 시 만료 여부를 확인하고 만료된 경우 차단 | 링크 유효기간 관리 필수 | 스터디 일지 열람 페이지, 만료/오류 페이지 |

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |

## 메뉴 구조

```
랜딩 페이지 (공개)
└── 서비스 소개 + "어드민 로그인" 버튼 - F010

어드민 링크 생성 페이지 (패스코드 인증 후)
├── Notion 일지 목록 선택 - F005
├── 만료일 설정 - F005
└── 링크 생성 및 복사 - F005, F010

스터디 일지 열람 페이지 (토큰 기반 공개)
├── 일지 내용 표시 - F001, F002, F003
└── PDF 다운로드 버튼 - F004

만료/오류 페이지 (자동 리디렉션)
└── 오류 상태 안내 - F003, F011
```

## 페이지별 상세 기능

### 랜딩 페이지

> **구현 기능:** `F010` | **접근:** 공개 (인증 불필요)

| 항목 | 내용 |
|------|------|
| **역할** | 서비스 소개 및 어드민 진입점 제공 |
| **진입 경로** | 도메인 루트 직접 접근 |
| **사용자 행동** | 서비스 소개를 읽고 어드민이면 로그인 버튼 클릭 |
| **주요 기능** | 서비스 설명 텍스트 및 특징 나열, **"어드민 로그인"** 버튼 (어드민 링크 생성 페이지 이동 트리거) |
| **다음 이동** | "어드민 로그인" 클릭 → 어드민 링크 생성 페이지 |

---

### 어드민 링크 생성 페이지

> **구현 기능:** `F005`, `F010` | **인증:** 환경변수 패스코드 검증 필요

| 항목 | 내용 |
|------|------|
| **역할** | Notion 일지를 선택하고 공유 URL(토큰)을 생성하는 어드민 전용 작업 페이지 |
| **진입 경로** | 랜딩 페이지의 "어드민 로그인" 버튼 클릭 후 패스코드 입력 성공 |
| **사용자 행동** | 패스코드 입력 → Notion DB에서 일지 선택 → 만료일 지정 → 링크 생성 → URL 복사 |
| **주요 기능** | 패스코드 입력 폼(환경변수 비교 검증), Notion DB 일지 목록 드롭다운 선택, 만료일 날짜 입력, **"링크 생성"** 버튼, 생성된 /log/[token] URL 표시 및 클립보드 복사 버튼, 링크 생성 시 Shared Links DB에 기록 자동 저장(F006) |
| **다음 이동** | 패스코드 오류 → 오류 메시지 표시(동일 페이지), 링크 생성 성공 → URL 표시(동일 페이지 유지) |

---

### 스터디 일지 열람 페이지

> **구현 기능:** `F001`, `F002`, `F003`, `F004` | **인증:** 없음 (토큰 기반 공개 접근)

| 항목 | 내용 |
|------|------|
| **역할** | 클라이언트가 토큰 URL로 접근하여 스터디 일지를 열람하고 PDF로 저장하는 핵심 페이지 |
| **진입 경로** | 어드민이 공유한 /log/[token] URL 직접 접근 |
| **사용자 행동** | URL 접근 → 토큰·만료일 자동 검증 → 일지 내용 열람 → 선택적으로 PDF 다운로드 |
| **주요 기능** | 토큰 유효성 및 만료일 자동 검증, Notion API로 일지 데이터 조회 및 렌더링(날짜/주제/내용/참석자/과제 필드), **"PDF 다운로드"** 버튼(@react-pdf/renderer 또는 Puppeteer 기반 생성) |
| **다음 이동** | 토큰 무효·만료 → 만료/오류 페이지 자동 리디렉션, PDF 다운로드 → 파일 저장 후 동일 페이지 유지 |

---

### 만료/오류 페이지

> **구현 기능:** `F003`, `F011` | **접근:** 자동 리디렉션

| 항목 | 내용 |
|------|------|
| **역할** | 토큰 만료, 토큰 무효, Notion API 오류 등 비정상 접근 상태를 안내하는 전용 페이지 |
| **진입 경로** | 스터디 일지 열람 페이지에서 검증 실패 시 자동 리디렉션 |
| **사용자 행동** | 오류 안내 메시지 확인 후 담당자에게 문의 또는 브라우저 종료 |
| **주요 기능** | 오류 유형별 메시지 표시(만료됨/존재하지 않는 링크/서버 오류), 오류 코드 표시, 랜딩 페이지 이동 링크 |
| **다음 이동** | "홈으로" 링크 클릭 → 랜딩 페이지 |

## 데이터 모델

### Notion 데이터베이스 스키마

> Notion이 유일한 데이터 저장소이며, 아래는 Notion DB 속성 필드 기준입니다.

#### Study Logs DB (`NOTION_DB_ID`)

| 필드명 | Notion 속성 타입 | 설명 |
|--------|----------------|------|
| 이름 | Title | 일지 제목 (예: "2주차 - React Hook 심화") |
| 날짜 | Date | 스터디 진행 날짜 |
| 태그 | Multi-select | 주제/카테고리 태그 |
| (본문) | Page Blocks | 상세 일지 내용 (heading, paragraph, code, callout 등 블록) |

#### Shared Links DB (`NOTION_LINKS_DB_ID`)

> 어드민이 링크를 생성할 때마다 자동으로 행이 추가됩니다 (F006).

| 필드명 | Notion 속성 타입 | 설명 |
|--------|----------------|------|
| 이름 | Title | 일지 제목 (발급 시 복사) |
| 일지ID | Rich Text | Study Logs DB 페이지 ID |
| 토큰 | Rich Text | 발급된 JWT 공유 토큰 |
| 만료일 | Date | 링크 만료일 |

## 기술 스택 (최신 버전)

### 프론트엔드 프레임워크

- **Next.js 15** (App Router) - React 풀스택 프레임워크, 서버 컴포넌트로 Notion API 호출
- **TypeScript 5.6+** - 타입 안전성 보장
- **React 19** - UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** (설정 파일 없는 새로운 엔진) - 유틸리티 CSS 프레임워크
- **shadcn/ui** - 고품질 React 컴포넌트 (Form, Button, Calendar, Badge 등)
- **Lucide React** - 아이콘 라이브러리

### 폼 & 검증

- **React Hook Form 7.x** - 패스코드 입력 폼 및 링크 생성 폼 상태 관리
- **Zod** - 패스코드·만료일 스키마 검증

### 데이터 소스

- **@notionhq/client** - Notion API 클라이언트 (DB 쿼리, 페이지 조회)
- Notion 데이터베이스를 유일한 데이터 저장소로 사용 (별도 DB 없음)

### PDF 생성

- **@react-pdf/renderer** (우선) - React 컴포넌트 기반 PDF 생성, 서버/클라이언트 양쪽 지원
- **Puppeteer** (대안) - HTML을 PDF로 변환, 복잡한 레이아웃에 유리

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼, 환경변수 관리

### 패키지 관리

- **npm** - 의존성 관리
